# Microsoft Copilot Fundamentals for Knowledge Workers

## Course Summary
This course provides knowledge workers with the skills to effectively use Microsoft Copilot within Microsoft 365 applications. It focuses on understanding Copilot's integration, crafting effective prompts, and ensuring responsible use through verification and adherence to company policies.

## Target Audience
Non-technical knowledge workers across Operations, Marketing, Finance, and HR, as well as team leads and managers.

---

# Curriculum Architecture

## Module M1: Introduction to Microsoft Copilot

### Summary
This module introduces Microsoft Copilot, its integration with Microsoft 365, and the data sources it utilizes.

### Objectives
- Identify what Microsoft Copilot is and how it integrates with Microsoft 365
- Explain the data sources Copilot uses

### Key Concepts
- Microsoft Copilot overview
- Integration with Microsoft 365
- Data sources (tenant graph data, LLM training data)
- Grounding concept
- Data boundary

### Activities
- Watch a demo video of Copilot summarizing a meeting transcript
- Participate in a discussion forum about data privacy

### Checks
- Type: mcq
  Description: What are the two main data sources for Copilot?

- Type: short_answer
  Description: Explain the concept of grounding in Copilot.

## Module M2: Effective Prompting with the CLEAR Framework

### Summary
This module teaches learners how to use the CLEAR framework to create effective prompts and understand the impact of prompt quality.

### Objectives
- Apply the CLEAR framework to create effective prompts
- Analyze the impact of prompt quality on output

### Key Concepts
- CLEAR framework (Context, Length, Examples, Audience, Refinement)
- Prompt quality
- Copilot modes (Draft, Transform, Summarize)

### Activities
- Practice creating prompts using the CLEAR framework
- Compare and contrast good vs. bad prompts

### Checks
- Type: mcq
  Description: Which element is NOT part of the CLEAR framework?

- Type: short_answer
  Description: Describe how prompt quality affects Copilot's output.

## Module M3: Responsible Use and Verification

### Summary
This module focuses on the responsible use of Copilot, emphasizing verification processes and policy adherence.

### Objectives
- Demonstrate the verification process for AI-generated content
- Identify scenarios that violate the Microsoft 365 Acceptable Use Policy

### Key Concepts
- Verification checklist
- Hallucination in AI
- Acceptable Use Policy
- Confidentiality creep
- Over-trust of fluency

### Activities
- Role-play scenario to practice verification
- Case study analysis of policy violations

### Checks
- Type: mcq
  Description: What is the first step in the verification checklist?

- Type: short_answer
  Description: Provide an example of a policy violation involving Copilot.
